### 1.2 Unhappy Path (Проблемные сценарии)

#### ⚠️ Проблема #1: Много "мусорных" слов
**Описание:** Пользователь пишет контекст, который не относится к трате

| Ввод пользователя | Что происходит сейчас | Проблема |
|------------------|----------------------|----------|
| `я играл с друзьями в CS2 после универа 150` | Парсится сумма 150, description содержит весь текст | Все слова попадают в ключевые слова категории |
| `встретился с другом в кофейне на работе 300` | amount: 300, слова "другом", "работе" → keywords | Захламление ключевых слов |
| `купил кофе после встречи на вокзале 200` | amount: 200, "встречи", "вокзале" → keywords | Нерелевантные ключевые слова |

**Текущее решение в коде:**
```python
# bot/services/expense_categorization.py
# Функция learn_keywords_on_create_sync фильтрует слова:
# - Минимум 3 символа
# - Исключает стоп-слова
# - Исключает слова-числа
```

**Что НЕ учтено:**
- ❌ Слова после суммы (контекст места/времени)
- ❌ Союзы и предлоги ("после", "перед", "около")
- ❌ Местоимения ("я", "мне", "моей")
- ❌ Слова из других категорий ("универ" → образование, но трата → кафе)

**Причина:**
```python
# Доходы определяются ТОЛЬКО по знаку + или словам "плюс"/"plus"
```

❌ **ПОТЕНЦИАЛЬНАЯ ПРОБЛЕМА:** Пользователи могут не знать про знак +

| дд.мм.гг | `15.03.24` | date(2024, 3, 15) (2000 + год) |

### 6.2 Unhappy Path дат

#### ⚠️ Проблема #9: Текстовые даты

|------------------|------------------|----------|
| `Кофе 200 вчера` | ❌ НЕ парсится | Нет поддержки |
| `Обед 500 позавчера` | ❌ НЕ парсится | Нет поддержки |

**Причина:**
```python

# Текстовые даты (yesterday, вчера, etc.) НЕ поддерживаются
```

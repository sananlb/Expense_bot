# Аудит скрытых команд бота

## Обзор проблемы

При анализе ошибки с командой `/report` обнаружено, что в боте есть команды с обработчиками, но которые **НЕ отображаются в меню** для пользователей. Это создаёт риски:
1. Пользователи могут случайно найти команду (через FAQ, подсказки) и получить ошибку
2. Мёртвый код усложняет поддержку
3. Дублирование функционала

---

### 4. `/categories` в category.py:10

**Статус:** ⚠️ **ДУБЛИКАТ - КРИТИЧНО**

**Что делает:**
```python
# Файл: bot/routers/category.py
@router.message(Command("categories"))
async def cmd_categories(message: types.Message):
    """Команда /categories"""
    # TODO: Показать список категорий пользователя
    await message.answer("🚧 Раздел категорий в разработке")
```

**Проблема:**
Есть ДВА файла с обработчиком `/categories`:
1. `category.py` - **ЗАГЛУШКА** с TODO, отвечает "🚧 Раздел категорий в разработке"
2. `categories.py` - **РАБОЧИЙ** обработчик с полным функционалом

**Какой используется:**
В `__init__.py`: `from .categories import router as category_router`
Значит используется **categories.py** (рабочий).

Но файл `category.py` существует и может создать путаницу!

**Рекомендация:** 🗑️ **УДАЛИТЬ файл category.py целиком** - это мёртвый код/заглушка.

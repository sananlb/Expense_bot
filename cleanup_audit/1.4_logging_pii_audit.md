❌ **Проблема:** Логируется доход пользователя с суммой

## 🔒 AI Промпты и PII

### Проверка отправки данных в OpenAI/Google AI

**Что отправляется в AI:**
1. ✅ Текст расхода (описание товара/услуги) - необходимо для категоризации
2. ✅ Сумма и валюта - необходимо для контекста
3. ✅ Список категорий пользователя - необходимо для выбора
4. ✅ Недавние категории (только названия) - для улучшения точности

**Что НЕ отправляется:**
- ✅ `telegram_id` - НЕ попадает в промпты
- ✅ `user_id` - используется только для внутреннего учёта метрик
- ✅ Имя пользователя - НЕ собирается
- ✅ Номер телефона - НЕ собирается
- ✅ Email - НЕ собирается

**Проверено в файлах:**
- `bot/services/ai_base_service.py:58-113` - Промпт категоризации
- `bot/services/openai_service.py:80-119` - Отправка в OpenAI
- `bot/services/google_ai_service.py:89-109` - Отправка в Google AI

**Код промпта:**
```python
def get_expense_categorization_prompt(
    self,
    text: str,
    amount: float,
    currency: str,
    categories: List[str],
    user_context: Optional[Dict[str, Any]] = None
) -> str:
    # ...
    return f"""You are an expense categorization assistant...

Expense information:
Description: "{text}"
Amount: {amount} {currency}

User's available categories:
{categories_list}
"""
```

✅ **Вывод:** Никакие PII не отправляются в AI промпты. Только необходимые данные для категоризации.

**Использование user_id:**
- `user_id` передаётся в `user_context` только для:
  - Записи метрик в БД (`AIServiceMetrics`)
  - Кеширования ответов
  - Rate limiting
- НЕ включается в текст промпта для AI

## ✅ Хорошие практики (обнаружено в коде)

### 1. Контекстное логирование
```python
logger.info(f"[AFFILIATE] Created referral: {referrer_id} -> {new_user_id}")
logger.info(f"Created category '{name}' (id: {category.id}) for user {user_id}")

```
✅ Включает префиксы модулей, ID сущностей, user_id

### 2. Разделение уровней логирования
- `logger.debug()` - детальная отладка (категоризация AI)
- `logger.info()` - важные события (создание/удаление)
- `logger.warning()` - потенциальные проблемы (лимиты)
- `logger.error()` - ошибки с `exc_info=True`

### 3. Структурированное логирование
```python
logger.info(f"Analytics query from user {user_id}: entity={spec.entity}, group_by={spec.group_by}")
```
✅ Позволяет легко парсить логи

### 4. Rate limiting логов
Не логируется каждый запрос к AI - только важные события (лимиты, ошибки)
